# 官方 Metal 示例研究结论

## 文档状态
- 当前有效（方法论参考）

## 核心结论
1. 公开 SDK 的硬件 RT 主流程是“加速结构 + compute kernel”，而不是独立 RT pass encoder。
2. 官方示例普遍使用 TLAS/BLAS、资源缓冲、可见函数/交点函数表来组织数据与调度。
3. 当前项目应优先对齐该模式，先完成硬件主线闭环，再恢复软件回退。

## 对本项目的直接要求
1. 以 compute pipeline 为单一主线完成 Stage 3D。
2. 维持场景/材质/资源布局一致性，为 Stage 4 回退复用做准备。
3. 明确能力检查与错误报告，避免隐式降级。

## 当前落地状态
- 已落地：TLAS/BLAS + compute kernel + 资源上传 + 基本 shading。
- 待落地：折射路径、交互累积性能、自动回归基线。
